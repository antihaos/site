Подробный конспект лекции для направления "Программная инженерия"

1. Введение: Проблема объема изображений

Проблема больших объемов несжатых изображений

Цифровые изображения представляют собой матрицы пикселей, каждый из которых содержит информацию о цвете. Без сжатия эти данные занимают значительный объем, что делает их хранение и передачу неэффективной.

для передачи/хранения] C --> D[Решение:
сжатие данных] subgraph Расчет_размера[Пример расчета 12МП фото] E["4000×3000 пикселей × 3 байта (RGB)"] E --> F["12,000,000 × 3 = 36 МБ"] end

Детальный расчет:

  • Разрешение 12 МП (4000×3000): 12,000,000 пикселей
  • Глубина цвета 24 бита: 3 байта на пиксель (RGB)
  • Без сжатия: каждый пиксель хранится независимо
Расчет размера несжатого изображения

где:

  • Ширина - количество пикселей по горизонтали
  • Высота - количество пикселей по вертикали
  • Биты на пиксель - глубина цвета (обычно 24 бита для RGB)
  • Деление на 8 - преобразование битов в байты
Пример расчета для 12МП фото

Для 12МП фото (4000×3000, 24 бита) размер несжатого изображения составляет около 36 МБ. Это объясняет необходимость сжатия для эффективного хранения и передачи.

Практические последствия:

  • Фотогалерея (1000 фото): ~36 ГБ
  • Веб-страница с 10 изображениями: ~360 МБ
  • Мобильное приложение: ограниченный трафик и хранилище

Математические основы:

Энтропия изображения

где \( p_i \) - вероятность появления i-го значения пикселя. Энтропия измеряет среднее количество информации на пиксель и определяет теоретический предел сжатия без потерь.

Корреляция между пикселями

где \( I(x,y) \) - значение пикселя в позиции (x,y), \( \mu \) - среднее значение изображения. Высокая корреляция указывает на пространственную избыточность.

Типы избыточности:

  1. Пространственная: соседние пиксели часто имеют похожие значения
  2. Спектральная: цветовые каналы (R, G, B) коррелированы между собой
  3. Психовизуальная: человеческий глаз не воспринимает некоторые детали
  4. Кодировочная: неоптимальное кодирование значений пикселей
Ключевые моменты:
  • Несжатое 12МП фото занимает около 36 МБ
  • Пространственная избыточность возникает из-за похожести соседних пикселей
  • Спектральная избыточность связана с корреляцией между цветовыми каналами
  • Психовизуальная избыточность позволяет удалять незаметные детали
  • Энтропия определяет теоретический предел сжатия без потерь

3. Методы сжатия без потерь (Lossless)

RLE (Run-Length Encoding)

Алгоритм RLE

где \( n \) - количество повторений, \( value \) - повторяющееся значение. RLE эффективен для изображений с большими областями одинакового цвета.

Пример: Последовательность "AAAAABBBBCC" кодируется как "5A4B2C"

Эффективность: Зависит от количества последовательных повторений

Применение: Простые изображения, иконки, скриншоты

def rle_encode(data):
    encoding = []
    i = 0

    while i < len(data):
        count = 1
        while i + 1 < len(data) and data[i] == data[i + 1]:
            i += 1
            count += 1
        encoding.append((count, data[i]))
        i += 1

    return encoding

def rle_decode(encoding):
    decoded = []
    for count, value in encoding:
        decoded.extend([value] * count)
    return decoded

# Пример использования
original = ["A", "A", "A", "A", "A", "B", "B", "B", "B", "C", "C"]
encoded = rle_encode(original)
decoded = rle_decode(encoded)

print(f"Исходные данные: {original}")
print(f"Закодированные данные: {encoded}")
print(f"Раскодированные данные: {decoded}")
print(f"Совпадение: {original == decoded}")

Средняя длина кода

где \( p_i \) - вероятность появления символа \( i \), \( l_i \) - длина кода для символа \( i \). Средняя длина кода Хаффмана всегда больше или равна энтропии источника.

Эффективность кодирования

Эффективность кодирования Хаффмана показывает, насколько близко средняя длина кода приближается к теоретическому пределу (энтропии источника).

LZW (Lempel-Ziv-Welch)

LZW - это словарный метод сжатия, который строит таблицу (словарь) строк на лету во время кодирования.

Алгоритм LZW

LZW использует идею замены повторяющихся последовательностей символов ссылками на словарь. Эффективен для изображений с повторяющимися паттернами.

def lzw_compress(data):
    # Инициализация словаря
    dict_size = 256
    dictionary = {chr(i): i for i in range(dict_size)}

    w = ""
    result = []
    for c in data:
        wc = w + c
        if wc in dictionary:
            w = wc
        else:
            result.append(dictionary[w])
            dictionary[wc] = dict_size
            dict_size += 1
            w = c

    if w:
        result.append(dictionary[w])

    return result

def lzw_decompress(compressed):
    # Инициализация словаря
    dict_size = 256
    dictionary = {i: chr(i) for i in range(dict_size)}

    result = []
    w = chr(compressed.pop(0))
    result.append(w)

    for k in compressed:
        if k in dictionary:
            entry = dictionary[k]
        elif k == dict_size:
            entry = w + w[0]
        else:
            raise ValueError('Некорректный сжатый поток')

        result.append(entry)
        dictionary[dict_size] = w + entry[0]
        dict_size += 1
        w = entry

    return ''.join(result)

# Пример использования
original = "TOBEORNOTTOBEORTOBEORNOT"
compressed = lzw_compress(original)
decompressed = lzw_decompress(compressed.copy())

print(f"Исходная строка: {original}")
print(f"Сжатые данные: {compressed}")
print(f"Распакованная строка: {decompressed}")
print(f"Совпадение: {original == decompressed}")

4. Методы сжатия с потерями (Lossy)

Преобразование цветового пространства

RGB в YCbCr

где Y - яркостная компонента, Cb и Cr - цветоразностные компоненты. Это преобразование позволяет эффективно отделить информацию о яркости (важную для восприятия) от информации о цвете.

Субдискретизация цветности

Человеческое зрение более чувствительно к изменениям яркости, чем цвета. Субдискретизация цветности позволяет уменьшить объем цветовой информации без заметной потери качества.

Преобразование в YCbCr и субдискретизация цветности - это первый шаг во многих алгоритмах сжатия с потерями, включая JPEG.

Дискретное косинусное преобразование (DCT)

Двумерное DCT

где \( F(u,v) \) - коэффициент DCT с частотными индексами u и v, \( f(x,y) \) - значение пикселя в позиции (x,y) блока 8×8. DCT преобразует блок пикселей из пространственной области в частотную.

Обратное DCT (IDCT)

Обратное преобразование позволяет восстановить изображение из частотных коэффициентов. При сжатии с потерями некоторые высокочастотные коэффициенты обнуляются или квантуются грубо.

DCT преобразует блок пикселей из пространственной области в частотную, где энергия сигнала концентрируется в небольшом количестве низкочастотных коэффициентов. Это позволяет эффективно сжимать изображение, отбрасывая малозначимые высокочастотные компоненты.

Квантование

Квантование коэффициентов DCT

где \( F_Q(u,v) \) - квантованный коэффициент DCT, \( F(u,v) \) - исходный коэффициент DCT, \( Q(u,v) \) - значение из матрицы квантования. Квантование - основной источник потерь в JPEG.

Матрица квантования JPEG

Стандартная матрица квантования JPEG. Высокочастотные коэффициенты квантуются грубее, так как они менее важны для визуального восприятия.

Квантование - это процесс уменьшения точности коэффициентов DCT. Высокочастотные коэффициенты квантуются грубее, так как они менее важны для визуального восприятия. Это основной источник потерь в JPEG.

Зигзаг-сканирование и RLE

graph TD A["Квантованные коэффициенты DCT"] --> B["Зигзаг-сканирование"] B --> C["Длинные последовательности нулей"] C --> D["RLE кодирование"] D --> E["Эффективное сжатие"] subgraph Зигзаг[Зигзаг-порядок] F["(0,0) → (0,1) → (1,0) → (2,0) → (1,1) → ..."] end

После квантования большинство высокочастотных коэффициентов становятся нулевыми. Зигзаг-сканирование упорядочивает коэффициенты так, чтобы нулевые значения группировались вместе, что повышает эффективность RLE кодирования.

6. Вейвлет-сжатие и JPEG2000

Дискретное вейвлет-преобразование (DWT)

Одномерное DWT

где \( \psi \) - вейвлет-функция (материнский вейвлет), \( a \) - параметр масштаба, \( b \) - параметр сдвига. DWT раскладывает сигнал на различные частотные компоненты.

Быстрое вейвлет-преобразование

где \( h \) и \( g \) - фильтры низких и высоких частот соответственно. Быстрое вейвлет-преобразование реализуется с помощью банка фильтров.

Вейвлет-преобразование обеспечивает лучшее частотно-пространственное разрешение по сравнению с DCT. Оно позволяет анализировать сигнал на разных масштабах и локализовать особенности в пространственной области.

7. Современные форматы изображений

WebP

WebP
Lossy/Lossless

Разработчик: Google

Сжатие: На 25-35% лучше чем JPEG при том же качестве

Особенности: Поддержка прозрачности, анимации, прогрессивной загрузки

Применение: Веб-изображения, мобильные приложения

WebP использует технологию прогнозирования блоков, аналогичную VP8 видео-кодеку. Формат поддерживает как сжатие с потерями, так и без потерь, а также анимацию.

HEIC/HEIF

HEIC/HEIF
Lossy/Lossless

Разработчик: MPEG

Сжатие: На 50% лучше чем JPEG при том же качестве

Особенности: Поддержка HDR, последовательностей изображений, глубины

Применение: Мобильные устройства (iOS), современные камеры

HEIC (High Efficiency Image Container) использует технологию сжатия HEVC (H.265) для изображений. Формат поддерживает продвинутые функции, такие как HDR, последовательности изображений и глубину.

AVIF

AVIF
Lossy/Lossless

Разработчик: Alliance for Open Media

Сжатие: На 50% лучше чем JPEG при том же качестве

Особенности: Открытый стандарт, поддержка HDR, широкого цветового охвата

Применение: Веб-изображения, современные приложения

AVIF (AV1 Image File Format) использует технологию сжатия AV1. Это открытый стандарт, обеспечивающий высокую эффективность сжатия и поддержку современных функций, таких как HDR и широкий цветовой охват.

Пример реализации DCT на Python

import numpy as np
import cv2
from scipy.fftpack import dct, idct

def dct_2d(block):
    """2D DCT преобразование блока 8x8"""
    return dct(dct(block.T, norm='ortho').T, norm='ortho')

def idct_2d(block):
    """2D обратное DCT преобразование"""
    return idct(idct(block.T, norm='ortho').T, norm='ortho')

def jpeg_quantization_table(quality=50):
    """Генерация матрицы квантования JPEG"""
    # Стандартная матрица квантования для яркости
    base_table = np.array([
        [16, 11, 10, 16, 24, 40, 51, 61],
        [12, 12, 14, 19, 26, 58, 60, 55],
        [14, 13, 16, 24, 40, 57, 69, 56],
        [14, 17, 22, 29, 51, 87, 80, 62],
        [18, 22, 37, 56, 68, 109, 103, 77],
        [24, 35, 55, 64, 81, 104, 113, 92],
        [49, 64, 78, 87, 103, 121, 120, 101],
        [72, 92, 95, 98, 112, 100, 103, 99]
    ], dtype=np.float32)

    if quality <= 0:
        quality = 1
    if quality > 100:
        quality = 100

    scale_factor = 5000 / quality if quality < 50 else 200 - 2 * quality
    quant_table = np.floor((base_table * scale_factor + 50) / 100)
    quant_table = np.clip(quant_table, 1, 255)

    return quant_table

def jpeg_compress_block(block, quality=50):
    """Сжатие блока 8x8 по алгоритму JPEG"""
    # DCT преобразование
    dct_block = dct_2d(block.astype(np.float32) - 128)

    # Квантование
    quant_table = jpeg_quantization_table(quality)
    quantized = np.round(dct_block / quant_table)

    return quantized

def jpeg_decompress_block(quantized, quality=50):
    """Декомпрессия блока 8x8"""
    # Обратное квантование
    quant_table = jpeg_quantization_table(quality)
    dct_block = quantized * quant_table

    # Обратное DCT
    block = idct_2d(dct_block) + 128
    return np.clip(block, 0, 255).astype(np.uint8)

Формулы для практических расчетов

PSNR (Peak Signal-to-Noise Ratio)

где \( MAX \) - максимально возможное значение пикселя, \( MSE \) - среднеквадратичная ошибка между исходным и сжатым изображением. PSNR широко используется для объективной оценки качества сжатия.

SSIM (Structural Similarity)

где \( \mu_x, \mu_y \) - средние значения, \( \sigma_x, \sigma_y \) - стандартные отклонения, \( \sigma_{xy} \) - ковариация, \( C_1, C_2 \) - константы стабилизации. SSIM лучше соответствует человеческому восприятию, чем PSNR.

Эффективность сжатия

Эффективность сжатия показывает, насколько хорошо алгоритм использует доступный битрейт для сохранения качества изображения.

from PIL import Image
import os

def compress_image(input_path, output_path, quality=85, format='JPEG'):
    """Сжатие изображения с заданным качеством"""
    with Image.open(input_path) as img:
        # Конвертация в RGB если необходимо
        if img.mode in ('RGBA', 'LA', 'P'):
            rgb_img = Image.new('RGB', img.size, (255, 255, 255))
            rgb_img.paste(img, mask=img.split()[-1] if img.mode == 'RGBA' else None)
            img = rgb_img

        # Сохранение с сжатием
        img.save(output_path, format=format, quality=quality, optimize=True)

def compare_image_sizes(original_path, compressed_path):
    """Сравнение размеров исходного и сжатого изображения"""
    original_size = os.path.getsize(original_path)
    compressed_size = os.path.getsize(compressed_path)

    compression_ratio = original_size / compressed_size
    space_saving = (1 - compressed_size / original_size) * 100

    return {
        'original_size': original_size,
        'compressed_size': compressed_size,
        'compression_ratio': compression_ratio,
        'space_saving': space_saving
    }

def convert_to_webp(input_path, output_path, quality=80):
    """Конвертация изображения в WebP формат"""
    with Image.open(input_path) as img:
        img.save(output_path, 'WEBP', quality=quality)

# Пример использования
if __name__ == "__main__":
    # Пример сжатия JPEG
    input_image = "example.jpg"
    output_jpeg = "compressed.jpg"
    output_webp = "converted.webp"

    # Сжатие JPEG
    compress_image(input_image, output_jpeg, quality=75)

    # Конвертация в WebP
    convert_to_webp(input_image, output_webp, quality=75)

    # Сравнение результатов
    jpeg_stats = compare_image_sizes(input_image, output_jpeg)
    webp_stats = compare_image_sizes(input_image, output_webp)

    print("Результаты сжатия:")
    print(f"JPEG: Коэффициент сжатия {jpeg_stats['compression_ratio']:.2f}:1")
    print(f"WebP: Коэффициент сжатия {webp_stats['compression_ratio']:.2f}:1")
    print(f"Экономия места JPEG: {jpeg_stats['space_saving']:.1f}%")
    print(f"Экономия места WebP: {webp_stats['space_saving']:.1f}%")
